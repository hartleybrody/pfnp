Getting started with forms

Whenever you want to collect information from your users, you'll do it with a <form> element and a series of one or more <input> elements nested inside. A form allows users to make a request to our "server" -- similar to clicking a link -- except that the request will also send along some extra information, based on what the user entered into the form.

At a high level, the <form> element describes the behavior of the form and what should happen when a user tries to submit it. The <input> elements describe each of the fields of the form, as well as the "submit" button.

Start Exploring

To the left is a very simple form, lightly styled with some bootstrap classes.

  1. Inspect each of the input fields in the form, both using your browser's developer tools and in the HTML file for this page.
  2. Inspect the outer form element. What does it tell you about how it will send the data to the server when a user clicks "submit"?
  3. Take a look at each of the <input> elements, and note the type and name attributes for each.
  4. Open the network tab in Chrome's developer tools, click the "Preserve Log" checkbox, and then submit the form. What request method was used? How did the values you typed in get sent to the server?
